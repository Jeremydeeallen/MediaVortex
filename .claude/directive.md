## Outcome

`Core/Path/Path.py` survives property-based fuzzing at scale: the algebraic identities promised by the contract (`Core/Path/path.feature.md`) hold across millions of synthetic inputs, not just the 30 hand-picked examples shipped in Phase 1. A new Hypothesis-driven test file produces zero counterexamples in a 1,000,000-example run. The pinned 1M-run command is reproducible by any future session and is recorded in this directive's verification block.

## Why now

Phase 1 shipped a unit-test grid that proves each acceptance criterion for one or two representative inputs. That is necessary but not sufficient -- Hypothesis routinely finds bugs in code that passes 100% of hand-written tests, especially around boundary characters in `RelativePath`, prefix-overlap pathology in `FromLegacyString`, and Unicode quirks in normalization. The Path class is the foundation of every v2 directive that follows (Phases 3-10 all import it). Catching a defect now costs one directive; catching it after Phase 7 caller migration costs a multi-phase rollback.

3. **Equality reflexivity property holds.** For every constructible `Path` p generated by the strategy, `p == p` is True. Asserted as a Hypothesis property.

4. **Equality symmetry property holds.** For every pair `(a, b)` of constructible Paths, `(a == b) == (b == a)`. Hypothesis property.

5. **Equality transitivity property holds.** For every triple `(a, b, c)` of Paths where `a == b` and `b == c`, `a == c` holds. Hypothesis property (the strategy synthesizes equal triples by reusing the same `(StorageRootId, RelativePath)` tuple, not by Hypothesis chance).

6. **Hash consistency property holds.** For every pair `(a, b)` where `a == b`, `hash(a) == hash(b)`. Hypothesis property.

7. **JSON round-trip property holds.** For every constructible `Path` p, `Path.FromJsonDict(p.ToJsonDict()) == p`. Hypothesis property (asserts S2 round-trippability over the input space).

8. **FromLegacyString round-trip property holds.** For every synthetic `(StorageRootId, CanonicalPrefix, RelativeTail)` triple generated by the strategy, `Path.FromLegacyString(prefix + tail_in_prefix_shape, [{"Id": id, "CanonicalPrefix": prefix}])` returns a Path whose `(StorageRootId, RelativePath)` equals `(id, normalized_tail)`. The strategy covers UNC, Windows-drive, and POSIX prefix shapes (S6 synthetic zoo).

9. **Normalization idempotency property holds.** For every input `RelativePath` that the constructor accepts, constructing `Path(id, p.RelativePath)` (i.e., re-feeding the post-normalization string) yields a Path with byte-identical `RelativePath`. Re-normalization is a no-op.

10. **ParentDir/Join identity property holds.** For every non-root Path p, `p.ParentDir().Join(p.LastSegment()) == p`. Hypothesis property over the strategy (asserts C12 holds beyond the single hand-written case).

11. **SplitExt structural property holds.** For every Path p, let `(base, ext) = p.SplitExt()`. Then (a) `base.RelativePath + ext == p.RelativePath` (concatenation reconstructs the original), (b) `base.StorageRootId == p.StorageRootId` (same root), (c) if `ext != ""` then `ext.startswith(".")`, `"/" not in ext`, `"\\" not in ext`, and `"." not in ext[1:]` (single trailing extension, no internal dot). NOTE: SplitExt is NOT idempotent across stacked extensions ("a.tar.gz" → ("a.tar", ".gz")) -- it strips exactly one extension per call, matching `os.path.splitext` semantics.

## Engineering Calls Already Made

- **Defect-handling protocol.** If a property surfaces a counterexample, the loop is: (a) verify it is a real defect, not a property-statement bug; (b) open a one-line entry under `## Status -> ### Findings` in this directive describing the counterexample; (c) fix `Core/Path/Path.py` in this directive (R13 allows edits to existing files); (d) if the fix changes a documented D-decision, promote a feature.md edit at DELIVERING. Do NOT defer defects to a follow-up directive -- the program's stop condition for Phase 2 is "design defect that can't be fixed in scope", so any in-scope defect MUST be fixed here.

- **Strategy locality.** Hypothesis strategies live inline in `test_path_properties.py` unless the file exceeds 350 LOC. No premature extraction to a helpers module. If extraction triggers, the helper goes at `Tests/Unit/_path_strategies.py` (not under `Core/`) -- strategies are test infrastructure, not production code.

- **RelativePath strategy shape.** The text strategy uses `hypothesis.strategies.text(alphabet=...)` over an alphabet that includes ASCII letters, digits, spaces, dots, hyphens, underscores, and forward slashes. Backslashes are included to exercise D9 normalization. The strategy filters out (a) empty strings only when the property under test requires non-empty (root-path properties accept empty), (b) inputs that start with `/` or `\` (rejected by constructor), (c) inputs containing `..` segments (rejected by constructor). Filtering is preferred over `assume()` for strategies that produce many invalid inputs; `assume()` is preferred for rare invalidity.

- **StorageRootId strategy.** `integers(min_value=1, max_value=10_000)`. Booleans (`isinstance(x, bool)` True / `isinstance(x, int)` True in Python) are excluded by the strategy choice -- `integers()` does not produce booleans.

- **Synthetic StorageRoots zoo for S6.** Strategy generates one of three prefix shapes: UNC (`\\<host>\<share>\`), Windows-drive (`<letter>:\`), POSIX (`/<root_segment>/`). The host/share/root_segment subcomponents use a constrained alphabet (ASCII letters + digits) to avoid prefix-overlap pathology unrelated to the property under test.

- **1M-example command.** `py -m pytest Tests/Unit/test_path_properties.py -p no:randomly --hypothesis-seed=0 --hypothesis-show-statistics -o "addopts=" --override-ini="filterwarnings=" -x` with per-test `@settings(max_examples=1_000_000, deadline=None)` decorator applied via a `--hypothesis-profile=million` profile registered at module-import time. Fixed seed (0) makes runs reproducible. `-x` exits on first failure to surface the shrunk counterexample immediately.

- **Where the 1M run runs.** I9 dev workstation. The operator owns service control on I9 per `feedback_user_starts_webservice.md`; running pytest is not a service operation, so I run it directly here at VERIFYING.

### Findings

- **F1 (2026-06-04): ParentDir/Join identity breaks for drive-letter-prefix RelativePaths.** Counterexample (test_parentdir_join_identity, --hypothesis-seed=0): `P = Path(StorageRootId=1, RelativePath='A:')`. The constructor accepts `"A:"` as a RelativePath (passes the leading-`/`/`\` strip check and `..` check). `P.ParentDir()` returns the root (no `/` in `"A:"`). `P.LastSegment()` returns `"A:"`. `root.Join("A:")` then raises `PathError` because `_DRIVE_PREFIX_RX` matches. The constructor and Join are inconsistent: one accepts the drive-letter pattern as relative, the other rejects it as an absolute marker. Root cause: D9 enumerates `..` and leading `/`\\ as rejected forms but is silent on drive-letter prefixes; the constructor mirrors the spec literally while Join was written defensively. Fix: tighten the constructor to also reject drive-letter-prefix RelativePaths (`^[A-Za-z]:` pattern), strengthening D9 to make "RelativePath is truly relative" enforced at construction. Hypothesis strategy updated to filter drive-letter-shaped first segments so generated inputs stay constructible. After fix: all 9 properties pass at default profile; 1M sweep re-run.
