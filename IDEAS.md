- 2026-06-19 | Bitrate-aware Transcode gating to prevent transcode-of-already-efficient sources. Current `EstimatedSavingsMB` compares source size to projected output size (using profile `TargetVideoKbps + TargetAudioKbps * duration`), so a 332 MB 720p HEVC source (already ~2 Mbps, BPP 0.09) gets routed to Transcode when the canary 720p AV1 target is ~450 kbps -- savings clear the 150 MB threshold but the result is generation-loss (~80-85 VMAF vs the canary's validated 92.92 baseline on uncompressed sources). Three approach options: (a) `MinSourceVideoBitrateRatio` on `transcoderules` (global, tiny effort, OK short-term); (b) `MinSourceBpp` on `profiles` (per-profile, normalizes across resolutions, default ~0.04 bpp on canary tiers, scales as profile catalog grows -- preferred); (c) codec-bucketed thresholds (`MinRatioForH264` vs `MinRatioForEfficientCodecs`, overkill until empirical data exists). Preferred: ship `MinSourceBpp` on `profiles` next to existing `audiobitratekbps`, plumb through `EffectiveProfile`, branch in `TranscodeOperation.Apply` before the savings-MB rule, add Settings input. Validate default via canary VMAF run on 30 Rock S01E01 (BPP 0.09 source) first to confirm the 0.04 threshold lands files in the right buckets.
- 2026-06-19 | Compliance refactor: collapse 4-operation `IComplianceOperation` hierarchy + `OperationResult` + `OperationsNeeded` set + `WorkBucket` derivation into three independent pure checks (`VideoCheck`, `ContainerCheck`, `AudioCheck`) + one bucket lookup table. Decision becomes `bucket = LOOKUP[(video_ok, container_ok, audio_ok)]` where the table is `{(T,T,T): None, (T,T,F): 'AudioFix', (T,F,T): 'Remux', (T,F,F): 'Remux', (F,*,*): 'Transcode'}`. Eliminates the 4 Rules tables (replaced with `VideoRules`/`ContainerRules`/`AudioRules`), the `chk_compliance_consistency` constraint, and the `OperationsNeededCsv` column. **Invariant the refactor establishes:** nothing downstream of compliance reads anything but `WorkBucket` -- workers, dashboards, queue routing all consume bucket only. Adding/removing/retuning a criterion then never ripples beyond one check function + one rule column + one Settings input. Surfaces the 30 Rock S01E01 routing bug for free (BPP-aware video check returns True for already-efficient HEVC, file falls to Remux, audio gets fixed, video stream-copies). Out of scope for any current audio-vertical work; size estimate: ~1-2 day directive. Sequenced AFTER audio vertical closes to 100% (Phase 2 in 2026-06-19 sequencing convo).
