Audio Compliance Rules

System-wide policy (singleton row). Applies to every MediaFile regardless of profile. Per-scope overrides for individual libraries / folders / files live at /AudioNormalization.

Target integrated loudness (LUFS)
Where the audio cluster aims. Streaming-TV standard ~-23 to -24. YouTube/Spotify ~-14.
Target true peak (dBTP)
Ceiling for true peaks. -2 dBTP is conservative; -1 dBTP is broadcast-standard tighter.
Max overshoot for adaptive fallback (dB)
When LINEAR loudnorm would clip by less than this, fall to ADAPTIVE silently (limiter absorbs the overshoot).
Max overshoot for review (dB)
Beyond this overshoot, file defers to the operator review queue (true outliers).
Acceptable audio codecs (CSV)
Source codecs allowed. Anything else triggers transcode for codec.
Enable Dialog Boost track
Emit a second LRA-compressed track and flag it as the container default.
Default to English when multiple languages present
RankPreferredDefaultPolicy. When false, the source's disposition.default wins.
Preferred default language rank (CSV)
Ordered list. First match in source wins. Default 'eng,en'.
Enable speech-based language detection
When source has no language tags, run a VAD-based detector. Slower probe; off by default.
Video Compliance Rules

System-wide policy (singleton row). Applies to every MediaFile regardless of profile.

Acceptable video codecs (CSV)
Files with codec NOT in this list -> needs transcode
Estimated savings threshold (MB)
Predicted savings >= threshold -> needs transcode (overridden by MinSourceBpp if source already efficient)
MinSourceBpp
If source bits-per-pixel-per-second is below this, skip transcode (already efficient). Default 0.04.
MaxSourceBpp
If source BPP is above this, force transcode even when the codec is in the allowlist (wastefully encoded source). Default 0.20.
Prevent upscale
When source resolution < profile target, never propose transcode
Block when source resolution exceeds profile target
Triggers transcode for downscale candidates
